### 4.5 버그픽스 — SSH 패스워드 역직렬화 (1f04ce0)

**원인**: `System.Text.Json`의 `[JsonIgnore]`는 직렬화/역직렬화 양방향을 모두 차단
→ IPC 수신 시 `SshConnection.Password`가 항상 `null` → SSH 인증 수단 없음 오류

**수정**: `SshConnectRequest` 입력 전용 DTO 분리
- IPC 수신 시 Password/Passphrase 정상 역직렬화
- `SshConnection.[JsonIgnore]` 유지 → 상태 전송 시 프론트 노출 차단

### 9.4 SSRF 정책 완화 (`2cebf8d`)

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| loopback (127.x) | Release 빌드 차단 | **허용** |
| 사설 IP (10.x, 172.x, 192.168.x, 100.x) | 차단 | **허용** |
| 클라우드 메타데이터 (169.254.x.x) | 차단 | **차단 유지** |
| 차단 시 알림 | 없음 | 팝업 alert 표시 |

**근거**: 데스크탑 앱에서 사용자가 직접 입력한 URL을 사설 IP라는 이유로 차단하는 것은 과잉 조치. 실제 SSRF 위험은 클라우드 메타데이터 서비스만 해당.

### 9.5 Ollama Bearer 인증 지원 (`b645625`)

**수정**:
- `LocalLlmService.SetApiKey()` 신규 — `HttpClient.DefaultRequestHeaders`에 `Authorization: Bearer {key}` 적용
- `AiServiceManager.SetApiKey()` ollama 케이스 추가
- 프론트엔드 `handleApplySettings` / `runDetailedConnectionTrace`: Ollama도 API Key IPC 전송

### 9.8 연결 실패 시 대상 서버 진단 명령어 출력 (`43bb532`)

Ollama 연결 실패(`runOpenWebUiDiag && !state.isConfigured`) 시 트레이스 로그 하단에 자동 출력:

```
── 대상 서버 SSH 접속 후 확인 ──────────────────────────────
   ① 서비스 실행:  curl -s http://localhost:{PORT}/api/version
                   systemctl status ollama   │   ps aux | grep ollama
   ② 포트 리스닝: ss -tlnp | grep {PORT}
   ③ 외부허용(Ollama): OLLAMA_HOST=0.0.0.0 ollama serve
   ④ 방화벽:       sudo ufw status   │   firewall-cmd --list-ports
────────────────────────────────────────────────────────────
```

포트 번호는 `new URL(endpoint).port`로 자동 추출.
