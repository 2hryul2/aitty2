## 🐛 버그 수정

### 1. WPF "(응답 없음)" 해결 — CRITICAL

**파일**: `src/Aitty/Ipc/IpcHandler.cs` (L283)

**근본 원인**: `HandleAiStream()` 내 onChunk 콜백에서 `Dispatcher.Invoke()` (동기) 사용 → 매 청크마다 UI 스레드를 블로킹 → 초당 수십 회 반복으로 WPF 전체 먹통

```csharp
// 변경 전 — 동기 블로킹
_webView.Dispatcher.Invoke(() => _webView.CoreWebView2.PostWebMessageAsJson(...));

// 변경 후 — 비동기 (fire-and-forget)
_ = _webView.Dispatcher.InvokeAsync(() => _webView.CoreWebView2.PostWebMessageAsJson(...));
```

**비교**: 같은 파일 L411, L429의 `HandleAiAnalyzeSsh`, `HandleAiSuggestCommand`는 이미 `InvokeAsync` 사용 → 이 1곳만 누락.

## ✨ 신규 기능

### 4. Chat/CLI 탭 전환 UI (Claude Desktop 벤치마킹)

**핵심**: AI 터미널을 **💬 Chat** + **⌨️ CLI** 탭으로 분리. 양쪽 항상 마운트(xterm 상태 보존), `display: none`으로 전환.

| 파일 | 역할 |
|------|------|
| `webapp/src/types/chat.ts` (신규) | `ChatMessage` 인터페이스 |
| `webapp/src/components/ChatPanel.tsx` (신규) | React 채팅 버블 UI + ReactMarkdown |
| `webapp/src/components/CodeBlock.tsx` (신규) | 코드 하이라이팅 + [📋 Copy] + [▶ Run] |
| `webapp/src/styles/chat.css` (신규) | 탭, 버블, 코드블록, 입력바 스타일 |
| `webapp/src/components/AITerminal.tsx` (수정) | 탭 전환 + 채팅 입력바 |
| `webapp/src/hooks/useAITerminal.ts` (수정) | `chatMessages` 상태 + dual-write 동기화 |
| `webapp/src/App.tsx` (수정) | `sshConnected` prop 전달 |

**Chat 탭 기능**:
- 채팅 버블: user(우측, 녹색), assistant(좌측, 다크)
- Markdown 렌더링: `react-markdown` + `remark-gfm`
- 코드블록: `react-syntax-highlighter` (vscDarkPlus)
- [📋 Copy]: 클립보드 복사 + 2초 "Copied!" 피드백
- [▶ Run]: SSH 터미널에 명령어 직접 전송 (`ssh.shellWrite`)
- 자동 스크롤: 새 메시지 시 하단 이동
- Enter 전송, Shift+Enter 줄바꿈

**Dual-Write 패턴** (CLI ↔ Chat 실시간 동기화):
```typescript
const contentRef = { current: '' }
ai.stream(message, (chunk) => {
  // CLI: xterm 직접 출력
  if (animationDone) term.write(chunk)
  else chunkBuffer.push(chunk)
  // Chat: React state 업데이트
  contentRef.current += chunk
  setChatMessages(prev => prev.map(m => m.id === assistantId ? { ...m, content: contentRef.current } : m))
})
```

**의존성 추가**: `react-markdown@^9`, `remark-gfm@^4`, `react-syntax-highlighter@^15`

### 5. 위험 명령어 차단 ([▶ Run] 보안)

**파일**: `webapp/src/utils/commandSafety.ts` (신규)

3단계 안전 등급으로 [▶ Run] 버튼 동작 제어:

| 등급 | 패턴 수 | 예시 | UI |
|------|---------|------|----|
| 🔴 danger | 10개 | `rm -rf /`, `mkfs.*`, fork bomb | 빨간 🚫 + 안전 대안 표시 |
| 🟠 caution | 12개 | `kill -9 1`, `iptables -F`, `ifconfig down` | 주황 ⚠️ + 안전 대안 표시 |
| 🟡 warning | 5개 | `curl\|sh`, `fdisk /dev/sd*` | 노랑 ⚠ + 안전 대안 표시 |
| ✅ safe | — | `ls`, `df -h`, `ps aux` | ▶ Run 정상 실행 |

**동작**:
- 멀티라인: 각 줄 검사, 최고 위험 등급으로 전체 판정
- 주석(`#`) 라인 자동 제외
- Shell 언어만 검사 (bash/sh/zsh/shell/console/text)
- `useMemo`로 안전성 결과 캐싱
- 위험 명령어 → 버튼 disabled + 경고 배지 + 안전한 대안 명령어 가이드

### 6. Thinking Indicator (Chat 탭 대기 메시지)

**파일**: `webapp/src/components/ThinkingIndicator.tsx` (신규)

 ※ AI는 정확하지 않는 정보를 제공할 수 있습니다. 중요한 정보는 반드시 확인하세요!

**동작**:
1. 10개 랜덤 메시지 중 1개 선택

3. 완료 후 깜박이는 커서 표시
4. 첫 실제 청크 도착 시 자동 전환 (`isStreaming && !content` → ThinkingIndicator, 그 외 → ReactMarkdown)
