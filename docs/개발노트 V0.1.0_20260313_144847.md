## 2. `AITerminal.tsx` — 주요 리팩터링 5건

### 2-1. DEFAULT_ENDPOINT 환경변수화 (L11)

```ts
// 이전
const DEFAULT_ENDPOINT = 'http://172.16.1.103:11434'

// 이후
const DEFAULT_ENDPOINT = import.meta.env.VITE_DEFAULT_OLLAMA_ENDPOINT || 'http://127.0.0.1:11434'
```

- Endpoint 입력 placeholder도 `{DEFAULT_ENDPOINT}` 동적 반영 (L825)
- 미설정 시 로컬호스트(`127.0.0.1`) 폴백

---

### 2-2. startupTracePrinted 모듈 플래그 (L12)

```ts
let startupTracePrinted = false
```

**목적**: React StrictMode의 이중 마운트, HMR 리로드 시 시작 진단 로그가 중복 출력되는 버그 방지.
- 최초 1회만 `runDetailedConnectionTrace('Startup', ...)` 실행
- 이후 마운트 시 `writePromptRef.current()` 즉시 호출로 스킵

---

### 2-3. 함수 Ref 안전화 (L24-27, L560-563)

```ts
// 신규 ref 선언
const sendMessageRef               = useRef<...>(async () => {})
const handleBuiltinCommandRef      = useRef<...>(async () => false)
const writePromptRef               = useRef<...>(() => {})
const runDetailedConnectionTraceRef = useRef<...>(async () => false)

// 동기화 effect
useEffect(() => { sendMessageRef.current = sendMessage }, [sendMessage])
useEffect(() => { handleBuiltinCommandRef.current = handleBuiltinCommand }, [handleBuiltinCommand])
useEffect(() => { writePromptRef.current = writePrompt }, [writePrompt])
useEffect(() => { runDetailedConnectionTraceRef.current = runDetailedConnectionTrace }, [runDetailedConnectionTrace])
```

**목적**: xterm `useEffect([], [])` 내부의 `term.onData` 핸들러가 stale closure를 참조하는 문제 해결.
- 이전: `onData` 내에서 직접 `sendMessage`, `writePrompt` 호출 → 초기화 시점 함수 고정
- 이후: ref를 통해 항상 최신 함수 참조 보장

---

### 2-4. runDetailedConnectionTrace 신설 (L172-278)

기존 `loadEngineState()` 단순 호출 방식을 대체하는 **단계별 접속 진단 함수**.

```ts
const runDetailedConnectionTrace = useCallback(async (
  title: string,       // 'Startup' | 'Check' | 'Apply'
  applyConfig: boolean // true → 설정 적용 후 진단, false → 현재 상태만 진단
): Promise<boolean>
```

**진단 단계 구조** (Ollama 기준, 총 7단계 / API Key 제공자는 6단계):

```
=== LLM 접속 상세 로그 (Check) ===
[HH:MM:SS] [1/7] Provider 선택 확인: Ollama (Local)
   ✓ activeProvider=ollama
[HH:MM:SS] [2/7] Ollama Endpoint 적용: http://...
   ✓ Endpoint 적용 완료
[HH:MM:SS] [3/7] Open WebUI API 연동 진단          ← Ollama 전용
   ✓ diagnosis: success=true, isOpenWebUi=true, models=12
[HH:MM:SS] [4/7] 엔진 상태 조회 (ai.state)
   ✓ isConfigured=true, provider=ollama, engine=ollama
[HH:MM:SS] [5/7] 모델 목록 조회 (ai.models)
   ✓ 모델 12개 확인
[HH:MM:SS] [6/7] Provider 상태 조회 (ai.providers)
   ✓ provider 2개, active=ollama
[HH:MM:SS] [7/7] 최종 판정
   ✓ 연결 성공: http://... | model=qwen2.5-coder:7b
AI 로그 저장: C:\...\aitty_api_log_20260313.txt
```

**특징**:
- 터미널 출력과 동시에 plainText(ANSI 제거)를 `traceLines[]`에 누적
- `saveApiLog=true` 이면 `ai.saveApiLog()` IPC 호출 → 로그 파일 저장
- `stripAnsi()` 로 ANSI 이스케이프 제거 후 저장

**호출처 변경**:

| 함수 | 이전 | 이후 |
|------|------|------|
| `handleCheck` | `loadEngineState(true)` | `runDetailedConnectionTrace('Check', true)` |
| `handleApplySettings` | `loadEngineState(false)` | `runDetailedConnectionTrace('Apply', false)` |
| 시작 시 | 간단 상태 출력 | `runDetailedConnectionTraceRef.current('Startup', false)` |

**Open WebUI 진단** (`ai.openWebUiDiagnose`): Ollama/Open WebUI 구분 + 모델 수 확인, 진단 로그 라인별 출력

### 2-5. saveApiLog 기능 (L50-63, L856-863)

```ts
const [saveApiLog, setSaveApiLog] = useState<boolean>(() => {
  const stored = localStorage.getItem('aitty.saveApiLog')
  return stored === null ? true : stored === '1'  // 기본값: ON
})

useEffect(() => {
  localStorage.setItem('aitty.saveApiLog', saveApiLog ? '1' : '0')
}, [saveApiLog])
```

**UI**: Settings 패널 버튼 행 좌측에 체크박스 추가
```
[✓ 로그저장]  [Check]  [Apply]  [시스템 프롬프트]
```

- 체크 시: Check/Apply/Startup 시 접속 진단 로그를 파일로 저장
- localStorage 영속: 앱 재시작 후에도 설정 유지
