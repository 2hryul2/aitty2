## 4. 보안 취약점 조치 (c36e939 + 1f04ce0)

### 4.1 Critical

| ID | 취약점 | 수정 파일 | 조치 |
|----|--------|-----------|------|
| **C-1** | Path Traversal (CWE-22) | `KeyManagerService.cs` | `FileInfo.FullName` 정규화 + UNC 경로 차단 + 1MB/10줄 제한 |
| **C-2** | SSRF | `LocalLlmService.cs` | URI scheme 검증 + private/loopback IP 블록 + `#if DEBUG` 로컬 예외 |

### 4.2 High

| ID | 취약점 | 수정 파일 | 조치 |
|----|--------|-----------|------|
| **H-1** | API Key 평문 저장 | `SecureApiKeyStore.cs` (신규), `AiServiceManager.cs` | Windows DPAPI 암호화, per-session 32byte Entropy, Dispose 시 zeroing |
| **H-2** | DevTools 프로덕션 노출 | `MainWindow.xaml`, `MainWindow.xaml.cs` | XAML 제거 → `#if DEBUG` CodeBehind 동적 추가 |
| **H-3** | 환경변수 우회 (`AITTY_DEV`) | `MainWindow.xaml.cs` | 환경변수 분기 완전 제거, `#if DEBUG` 컴파일 타임 분기로 대체 |

### 4.3 Medium

| ID | 취약점 | 수정 파일 | 조치 |
|----|--------|-----------|------|
| **M-1** | IPC 입력 검증 부재 | `IpcHandler.cs` | 1MB 메시지 크기 제한 + `NotSupportedException` 타입명 은닉 |
| **M-2** | SSH 자격증명 메모리 잔류 | `SshConnection.cs`, `SshService.cs` | `[JsonIgnore]` + `IDisposable` + `Dispose()` 시 Password zeroing |
| **M-3** | SSH 명령 감사 로그 없음 | `SshAuditLogger.cs` (신규), `IpcHandler.cs` | NDJSON 감사 로그 (`%AppData%\ssh-ai-terminal\logs\audit_YYYYMMDD.log`) + 민감 패턴 마스킹 |
| **M-4** | 프론트엔드 민감 정보 로그 | `webapp/src/utils/logger.ts` | `SENSITIVE_KEYS` 마스킹 + Production `WARN` 레벨 필터 |

### 4.4 Low

| ID | 취약점 | 수정 파일 | 조치 |
|----|--------|-----------|------|
| **L-1** | `index.html` 반복 디스크 I/O | `MainWindow.xaml.cs` | 메모리 캐시(`byte[]`) 1회 로드 |
| **L-2** | SSH 키 경로 경계 검사 부재 | `KeyManagerService.cs` | `IsPathAllowed()` 경계 검사 포함 |

### 4.5 버그픽스 — SSH 패스워드 역직렬화 (1f04ce0)

**원인**: `System.Text.Json`의 `[JsonIgnore]`는 직렬화/역직렬화 양방향을 모두 차단
→ IPC 수신 시 `SshConnection.Password`가 항상 `null` → SSH 인증 수단 없음 오류

**수정**: `SshConnectRequest` 입력 전용 DTO 분리
- IPC 수신 시 Password/Passphrase 정상 역직렬화
- `SshConnection.[JsonIgnore]` 유지 → 상태 전송 시 프론트 노출 차단
