### 3. SSH Terminal 간헐적 입력 불가 수정 (`7b0fdab`)

- **증상**: SSH Terminal에서 간헐적으로 키보드 입력이 전혀 동작하지 않음
- **두 가지 독립적 원인** 확인:

#### 원인 A — ShellStream Thread Safety

```
[문제]
OnWebMessageReceived (async void) → 동시에 여러 인스턴스 실행 가능
  ├─ shellRead 폴링(100ms) → ReadFromShell() → _shellStream.Read()
  └─ 키 입력 → shellWrite → WriteToShell() → _shellStream.Write() + Flush()
두 경로가 lock 없이 동일 _shellStream 동시 접근 → SSH.NET ShellStream 비thread-safe → Write 유실 → 입력 무반응

[수정] src/Aitty/Services/SshService.cs
```

```csharp
private readonly object _streamLock = new();

public void WriteToShell(string data)
{
    lock (_streamLock) { _shellStream?.Write(data); _shellStream?.Flush(); }
    // lock 밖: Remember(), _lastOutputBuffer 등 로컬 버퍼 처리
}

public string? ReadFromShell()
{
    lock (_streamLock)
    {
        if (_shellStream is null || !_shellStream.CanRead) return null;
        if (!_shellStream.DataAvailable) return null;
        try { return _shellStream.Read(); }
        catch { return null; }
    }
}
```

#### 원인 B — xterm.js 포커스 소실

```
[문제]
- 접속 성공 후 term.focus() 호출 없음 → 초기 포커스 미설정
- AI Terminal 클릭, 패널 리사이저 드래그 등 UI 조작 후 포커스 소실

[수정] webapp/src/components/SSHTerminal.tsx
```

```typescript
// ① 접속 성공 직후
setTimeout(() => termRef.current?.focus(), 50)  // WebView2 렌더 후 포커스

// ② 컨테이너 클릭 시 포커스 복원
containerEl?.addEventListener('click', handleClick)
```

### 4. AI Terminal 회색 화면 + SSH 입력 불가 수정 (`596792f`)

- **증상**: LLM 스트리밍 출력 중 마우스 클릭 → 화면이 회색으로 변함 → SSH Terminal 키보드 입력 불가
- **두 가지 독립적 원인** 확인:

#### 원인 A — Typewriter 애니메이션의 과도한 canvas 리페인트 (PRIMARY)

```
[문제]
sendMessage() 내 waiting 배너 출력:
  4개 라인 × 평균 ~80문자 × 15ms/char = 320개 setTimeout 콜백 (~4.8초)
  각 term.write(char)마다 xterm.js canvas 전체 리페인트
  → 마우스 클릭 이벤트 수신 시 이벤트 루프 과부하
  → WebView2 GPU compositor 표면 무효화 → 회색 화면

[수정] AITerminal.tsx — handleAnalyzeClick / sendMessage 두 함수 모두
```

```typescript
// BEFORE: 1문자/15ms (320 setTimeout × canvas repaint)
for (const char of text) {
  term.write(char)
  await new Promise(r => setTimeout(r, 15))
}

// AFTER: 즉시 출력 (0 setTimeout)
term.write(`${color}${text}\x1b[0m\r\n`)
```

#### 원인 B — Streaming 종료 후 포커스 미복원 (SECONDARY)

```
[문제]
회색 화면 → WebView2 GPU 표면 재생성 시 포커스 초기화
sendMessage() finally 블록: setIsStreaming(false) 후 포커스 복원 없음
→ SSH Terminal이 클릭 전까지 키보드 입력 불가

[수정] AITerminal.tsx + SSHTerminal.tsx — CustomEvent 활용
```

```typescript
// AITerminal.tsx — 스트리밍 종료 시 이벤트 디스패치
} finally {
  isProcessingRef.current = false
  setIsStreaming(false)
  window.dispatchEvent(new CustomEvent('ai-streaming-end'))
}

// SSHTerminal.tsx — 이벤트 수신 시 포커스 복원
const handleStreamingEnd = () => term.focus()
window.addEventListener('ai-streaming-end', handleStreamingEnd)
// cleanup: window.removeEventListener('ai-streaming-end', handleStreamingEnd)
```

### 세션 복원 흐름
```
앱 종료:
  MainWindow.Closing → SessionService.Save(aiManager.GetSessionData())
  → %AppData%\ssh-ai-terminal\session.json 저장

앱 시작:
  MainWindow_Loaded → SessionService.LoadAsync()
  → IpcHandler 생성 시 restoredSession 전달
  → 프론트엔드 마운트: session.getRestored() IPC 호출
  → AITerminal: model / engine / systemPrompt 상태 동기화
  → 터미널에 "세션 복원" 메시지 출력
```
