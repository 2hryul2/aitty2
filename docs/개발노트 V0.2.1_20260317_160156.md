- [x] **한글 Backspace 잔상 버그 수정** (`4d418ec`)
  - **증상**: AI 터미널에서 한글 입력 후 Backspace 시 문자가 시각적으로 지워지지 않음 (영어는 정상)
  - **원인 분석** (3회 시도를 통해 확인):

  | 시도 | 접근 | 실패 원인 |
  |------|------|----------|
  | 1차 | `xterm-addon-unicode11` 추가 + `\b\b  \b\b` | 개발노트 V0.1.0 이력: WebView2 Release 검은 화면 유발 |
  | 2차 | `activeVersion` 초기화 순서 수정 | `term.unicode.wcwidth` = xterm.js v5 미공개 API → TypeError 묵음 실패 |
  | 3차 | `charDisplayWidth()` 직접 구현 + `\b\b  \b\b` | space 쓰기가 wide char 내부 spacer 셀 미정리 |
  | **4차** | `buffer.cursorX` + CSI D + **ECH** | **✅ 해결** |

  - **최종 해결책**:
    - `term.buffer.active.cursorX` → xterm.js 버퍼에서 실제 커서 열 조회 (추정 불필요)
    - `\x1b[WD` (CSI D): 커서 W칸 후진 (wide char = 2, 영어 = 1)
    - `\x1b[WX` (ECH): W셀 강제 소거 — wide char의 spacer 셀까지 완전 초기화
    - 줄 바꿈 케이스: `curX < w` 감지 → `\x1b[A` 이전 줄 이동 + 절대 열 위치 지정

  - **핵심 발견** — ECH vs space 쓰기 차이:
    ```
    space 쓰기:  Cell[N+1] = spacer 그대로 잔상 ❌
    ECH:        Cell[N+1] = default 초기화 완전 소거 ✅
    ```
  - **`xterm-addon-unicode11` 사용 금지** 재확인: V0.1.0 개발노트에서 이미 WebView2 Release 검은 화면 이력으로 제거됨 — 한글은 기본 UnicodeV6(`table.fill(2, 0xac00, 0xd7a4)`)에서 이미 2 cell 처리

## 🔧 주요 구현 로직

### SSH Idle Timeout 감지 흐름
```
[시나리오 1: sshd 서버 측 세션 만료]
sshd → SSH_MSG_CHANNEL_CLOSE 전송
    ↓
_shellStream.CanRead = false (즉시)
    ↓
ReadFromShell() → CanRead 체크 → null 반환
    ↓
최대 10초 후 헬스체크 실행
    ↓
ssh.state() → IsConnected → false
(_client.IsConnected=true여도 CanRead=false면 false)
    ↓
터미널: "⚠ SSH 연결이 끊어졌습니다." + 연결 폼 복원

[시나리오 2: 네트워크 끊김 / 서버 강제 종료]
TCP 연결 단절
    ↓
KeepAlive(15s) 패킷 실패 → _client.IsConnected = false
    ↓
최대 10초 후 헬스체크 감지 → 동일 처리
```

### 한글 Backspace 처리 흐름 (ECH 방식)
```
[일반 케이스 — 한글 "가" (2 cell), cursor at col 17]
  chars = [...'가 나']  →  pop()  →  lastChar = '나'
  w = charDisplayWidth('나') = 2
  curX = term.buffer.active.cursorX = 21
  curX(21) >= w(2) → 일반 케이스
  term.write('\x1b[2D\x1b[2X')
    ├─ \x1b[2D : cursor 21 → 19
    └─ \x1b[2X : ECH — cell[19], cell[20] 완전 초기화 (spacer 포함) ✅

[줄 바꿈 케이스 — "가"가 col 98-99, cursor at col 0 (다음 줄)]
  curX(0) < w(2) → 줄 바꿈 케이스
  targetCol = term.cols(100) - w(2) = 98
  term.write('\x1b[A\x1b[99G\x1b[2X')
    ├─ \x1b[A   : 이전 줄로 이동
    ├─ \x1b[99G : col 98 (0-indexed) 절대 이동
    └─ \x1b[2X  : cell[98], cell[99] 소거 ✅
```

### ECH vs space 쓰기 — xterm.js 내부 셀 처리 비교
```
xterm.js wide char 내부 표현:
  Cell[N]   = { char: '가', width: 2 }
  Cell[N+1] = { char: '',   width: 0 }  ← spacer (오른쪽 절반 플레이스홀더)

[space 쓰기: \b\b  \b\b — 실패]
  space(' ')를 Cell[N]에 쓰면:
    Cell[N]   = { char: ' ', width: 1 }  ✅
    Cell[N+1] = { char: '',  width: 0 }  ❌ spacer 그대로 → 오른쪽 잔상

[ECH: \x1b[2X — 성공]
  ECH는 N ~ N+1 셀 전체를 default 속성으로 초기화:
    Cell[N]   = { char: ' ', width: 1 }  ✅
    Cell[N+1] = { char: ' ', width: 1 }  ✅ spacer도 정리 → 완전 소거
```
