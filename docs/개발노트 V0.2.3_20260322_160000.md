### Phase 2: P0 — Critical 수정 (3항목)

#### P0-1~3: HttpClient `DefaultRequestHeaders` thread-safety

- **문제**: `ClaudeApiService`, `OpenAiService`, `LocalLlmService` 3개 서비스에서 `HttpClient.DefaultRequestHeaders`를 직접 조작 → 멀티스레드 환경에서 race condition
- **수정**: `DefaultRequestHeaders` 제거, 모든 HTTP 호출을 `HttpRequestMessage` + per-request header 패턴으로 변경

| 서비스 | 변경 전 | 변경 후 |
|--------|--------|--------|
| `ClaudeApiService.cs` | `_client.DefaultRequestHeaders.Add("x-api-key")` | `request.Headers.Add("x-api-key", _apiKey)` |
| `OpenAiService.cs` | `_client.DefaultRequestHeaders.Authorization = ...` | `request.Headers.Authorization = new(...)` |
| `LocalLlmService.cs` | `DefaultRequestHeaders.Remove/Add` | `AddAuthHeader(HttpRequestMessage)` 헬퍼 |

#### P0-4: SshService buffer race condition

- **문제**: `_lastOutputBuffer`, `_recentLines`, `_collectingOutput`가 여러 스레드에서 잠금 없이 접근
- **수정**: `private readonly object _bufferLock = new()` 추가, 모든 버퍼 접근을 `lock (_bufferLock)` 블록으로 보호

```csharp
// ConnectAsync, WriteToShell, ReadFromShell, GetLastCommandOutput,
// GetRecentOutput, Remember — 6개 메서드에 lock 적용
lock (_bufferLock)
{
    _lastOutputBuffer.Clear();
    _recentLines.Clear();
}
```

### Phase 5: PublishSingleFile 배포 수정 (v0.2.3 핵심)

#### 문제 1: wwwroot 파일이 SingleFile exe에 임베드됨
- **증상**: `Navigation error: ConnectionAborted`, `ERR_FILE_NOT_FOUND`
- **원인**: `PublishSingleFile=true`가 Content 파일을 exe 내부에 번들링 → `AppContext.BaseDirectory` 기준 `wwwroot/` 폴더가 비어있음
- **수정**: `ExcludeFromSingleFile=true` 추가

```xml
<Content Include="wwwroot\**\*">
  <CopyToPublishDirectory>PreserveNewest</CopyToPublishDirectory>
  <ExcludeFromSingleFile>true</ExcludeFromSingleFile>
</Content>
```

#### 문제 2: MSBuild ItemGroup 평가 시점
- **증상**: `dist/wwwroot/` 디렉토리 존재하지만 파일 없음 (빈 폴더)
- **원인**: 정적 `<ItemGroup Condition="Exists('wwwroot')">` 가 프로젝트 로드 시점에 평가됨. `BuildReactApp` 타겟이 나중에 wwwroot를 생성하지만, ItemGroup은 이미 `false`로 스킵된 상태
- **수정**: 동적 타겟으로 변경

```xml
<!-- 변경 전: 정적 ItemGroup (프로젝트 로드 시 평가) -->
<ItemGroup Condition="Exists('wwwroot')">...</ItemGroup>

<!-- 변경 후: 동적 타겟 (BuildReactApp 이후 평가) -->
<Target Name="IncludeWwwroot" AfterTargets="BuildReactApp" BeforeTargets="BeforeCompile">
  <ItemGroup>
    <Content Include="wwwroot\**\*" Condition="Exists('wwwroot')">
      <ExcludeFromSingleFile>true</ExcludeFromSingleFile>
    </Content>
  </ItemGroup>
</Target>
```

#### 문제 3: WebResourceRequested 핸들러 제거
- **변경 전**: `WebResourceRequested` 이벤트로 index.html 가로채기 + 메모리 캐시
- **변경 후**: 런타임에 `wwwroot/index.html` 파일 직접 수정(crossorigin 제거) + 순수 가상호스트 매핑

```csharp
// 런타임에 crossorigin 속성 제거 (1회성)
var html = await File.ReadAllTextAsync(indexPath, Encoding.UTF8);
if (html.Contains(" crossorigin"))
{
    html = html.Replace(" crossorigin", "");
    await File.WriteAllTextAsync(indexPath, html, Encoding.UTF8);
}

// 가상 호스트 등록 — 순수 매핑만 사용
webView.CoreWebView2.SetVirtualHostNameToFolderMapping(
    "app.local", wwwroot, CoreWebView2HostResourceAccessKind.Allow);
webView.CoreWebView2.Navigate("https://app.local/index.html");
```

#### 문제 4: `stopPolling` TDZ (Temporal Dead Zone) 에러
- **증상**: 검은 화면, `Cannot access 'stopPolling' before initialization`
- **원인**: `SSHTerminal.tsx`에서 `startPolling`의 `useCallback` deps에 `stopPolling` 참조 → `stopPolling`은 `const`로 아래에 선언 → TDZ 에러
- **수정**: `stopPolling` 선언을 `startPolling` 위로 이동

```typescript
// 변경 전 (TDZ 에러)
const startPolling = useCallback(() => {
  // ... stopPolling() 사용
}, [shellRead, stopPolling])  // ← stopPolling 아직 미선언

const stopPolling = useCallback(() => { ... }, [])  // ← 여기서 선언

// 변경 후 (정상)
const stopPolling = useCallback(() => { ... }, [])  // ← 먼저 선언

const startPolling = useCallback(() => {
  // ... stopPolling() 사용
}, [shellRead, stopPolling])  // ← 이미 선언됨
```

PublishSingleFile 배포 시 검은 화면 문제 해결에 사용한 단계적 진단 방법:

| # | 진단 방법 | 확인 결과 |
|---|----------|----------|
| 1 | `MessageBox`로 wwwroot 경로/파일 존재 여부 표시 | Dir exists: True, index.html: False (빈 폴더) |
| 2 | `ExecuteScriptAsync`로 페이지 상태 확인 | rootChildren: 0, scripts 로드됨, styleSheets: 1 |
| 3 | `fetch()` + `import()`로 JS 파일 검증 | status: 200, contentType: text/javascript, importOk |
| 4 | `ErrorBoundary` 컴포넌트 추가 | `Cannot access 'stopPolling' before initialization` |
| 5 | 비압축 빌드로 정확한 줄 번호 확인 | `SSHTerminal.tsx:18847` — const TDZ 에러 |

**교훈**: WebView2 환경에서 `window.onerror`는 module script 에러를 잡지 못함. React ErrorBoundary + 비압축 빌드 조합이 가장 효과적.
